# WebForge Agent Registry

This file lists every named agent in the WebForge organization. HR maintains this file. Only HR can add, activate, or deactivate agents.

**Status legend:**
- **Active** — currently working or available
- **Inactive** — registered but not currently assigned (name is kept forever)
- **Reserved** — name held for future use

## Department 0 — META ENGINEERING (5 agents)

**Director:** Daedalus

This department maintains WebForge itself. When bugs are found in MCPs, when new MCPs need to be built, when agents need to be created or upgraded — this department does it.

| Agent | Role | Status | Reports To |
|---|---|---|---|
| Daedalus | Meta Engineering Director | Active | Hermes |
| Forge | MCP Builder | Active | Daedalus |
| Anvil | MCP Fixer | Active | Daedalus |
| Loom | Agent Creator | Active | Daedalus |
| Compass | System Tester | Active | Daedalus |

### What this department does

- **Forge** — builds new MCPs when the system needs new capabilities
- **Anvil** — fixes bugs in existing MCPs (like the gen-002 summary bug I fixed manually)
- **Loom** — creates new agents when HR needs more named agents
- **Compass** — tests the whole WebForge system to find issues before they break projects
- **Daedalus** — leads the team, decides what to build/fix next

This department runs **after** a project completes — it reviews what went wrong and improves the system so the next project goes smoother.
