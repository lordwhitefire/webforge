# /odin

Wake the Odin Team to research standards.

## What this command does

1. Wakes Athena (Intelligence Director).
2. Athena wakes all 17 Odin agents (Odin-Sage through Odin-Aster).
3. Each Odin agent researches best practices for their 5 areas.
4. Each writes findings to memory with links to sources.
5. Athena sends a summary to the CEO.

## How to use

Type `/odin` AFTER:
- The Probe team has finished (`/probe`)
- You want to research standards before building

## What happens next

- Odin agents use the Standards MCP to fetch live docs.
- They might use the Web Search MCP for additional research.
- They write findings to memory.
- The CEO reviews the standards with you before Build starts.

## Difference from /probe

- `/probe` checks what's DECIDED and what's MISSING.
- `/odin` researches the BEST WAY to do what's decided.

## Example

```
> /odin
[Athena] Waking Odin Team...
[Odin-Sage] Researching standards for areas 01-05...
[Odin-Sage] Fetching Next.js docs via Standards MCP...
[Odin-Sage] Fetching Vercel deployment docs...
[Odin-Sage] Findings written to memory.
...
[Athena] All 17 Odin agents done. Summary sent to CEO.
```
